## OPE-19 Chat Display Tools — Deep Debugging (22:00–23:00 UTC)

### Two Bugs Found

**Bug 1: Missing X-Org-Id header (HTTP 400)**
- `DefaultChatTransport` in AI SDK v6 uses `headers: orgHeaders()` — evaluated ONCE at component mount
- If `localStorage.getItem('activeOrgId')` returns a value, the header IS set, but the transport may not be sending it
- Manual `fetch()` with explicit `X-Org-Id` header returns 200
- **Fix applied**: Changed `headers: orgHeaders()` → `headers: () => orgHeaders()` in `ChatWidget.tsx` on GB10
- Frontend rebuild completed, running on port 3000

**Bug 2: jsonSchema() serialization — `type: "None"` (THE REAL BLOCKER)**
- `jsonSchema()` from `ai` package wraps schemas as `{jsonSchema: {type: 'object', properties: {...}}}`
- But `@ai-sdk/openai@3.0.41` serializes it to OpenAI as `parameters: {properties: {}, additionalProperties: false}` — **strips `type` and actual properties**
- OpenAI rejects with: `Invalid schema for function 'get_overview': schema must be a JSON Schema of 'type: "object"', got 'type: "None"'`
- **Confirmed via fetch monkey-patch**: Logged actual HTTP request body sent to OpenAI — `tool[0].parameters` has empty properties
- This affects ALL 21 data tools + 3 display tools = 24 tools total
- `jsonSchema()` worked in isolation (`npx tsx -e` test) but NOT through the full `streamChatResponse` → `streamText` → OpenAI provider pipeline

### Root Cause Analysis
- The `ai@6.0.116` + `@ai-sdk/openai@3.0.41` + `zod@4.3.6` combination has a serialization bug
- `tool.inputSchema` is used by the OpenAI provider via `asSchema(inputSchema).jsonSchema` — this double-wraps and strips the type
- **The fix**: Convert all 21 tools back to `z.object({...})` using `import { z } from 'zod/v4'` which the AI SDK handles correctly internally
- This is a clean rewrite of tool parameter definitions, NOT a hack

### Key Learning
- `jsonSchema()` from `ai` package is BROKEN with `@ai-sdk/openai` when Zod v4 is installed — the provider's `asSchema()` doesn't properly unwrap the jsonSchema wrapper
- Always use `z.object()` with proper Zod types for AI SDK tools — never raw JSON Schema via `jsonSchema()`
- `zod/v4` import path works: `import { z } from 'zod/v4'`
- `next start` (production mode) DOES support `rewrites()` from next.config — earlier assumption it didn't was wrong
- `agent-browser` works for testing: `eval` for JS execution, `snapshot` for DOM state, `console` for errors
