- Spec written, Dex reviewed (6/10 first pass), Linus (Opus) + Marcus (Sonnet) built in parallel
- Dex re-reviewed: backend 4/10 → Linus on Opus fixed all 5 issues → 9/10
- **Root cause of widget not rendering**: ChatWidget was in `(dashboard)/layout.tsx` but home page (`/`) is `app/page.tsx` which doesn't use that layout. Fixed by moving to root `layout.tsx` via `ChatWidgetLoader.tsx` wrapper (client component with `dynamic(() => import(...), { ssr: false })` because root layout is server component).
- **`dynamic` naming conflict**: `export const dynamic = 'force-dynamic'` clashed with `import dynamic from 'next/dynamic'`. Renamed to `nextDynamic`, then switched to wrapper pattern.
- **Chat streaming 502 in dev mode**: Next.js dev proxy buffers SSE responses. Fixed by running production mode (`next build && next start`).
- **Chat "404 Page not found" error**: `@ai-sdk/openai` `createOpenAI` sends request format MiniMax rejects. Fixed by **bypassing AI SDK entirely** — raw `fetch` to MiniMax `/chat/completions` + manually emitting `text-start`/`text-delta`/`text-end` JSON event stream format that `DefaultChatTransport` expects.
- **Lesson**: AI SDK v6 `createOpenAI` is NOT compatible with MiniMax M1. Always test LLM integrations with raw fetch first before wrapping in SDK.
