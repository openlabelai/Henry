
## 2026-03-04 ~02:36 UTC — Spotify Integration & Security Fixes

### Major Features Built
1. **Spotify Play Count Integration**
   - Integrated sp-playcount API (unofficial Spotify partner API proxy)
   - `fetchArtistPlayCounts()` in catalogService.ts
   - Auto-fetches after artist link during onboarding
   - 24h cache via `playCountsFetchedAt`
   - Stores monthly listeners on CanonicalArtist

2. **Stream Discrepancy Forensic Rule**
   - New rule `stream-discrepancy` compares Spotify streams vs report streams
   - Thresholds: 10% info, 30% warning, 50% critical
   - Found ~$12K potentially unreported for BLANKO (30% gaps on multiple tracks)

3. **Catalog Preview UI**
   - `CatalogPreview.tsx` shows tracks after artist selection in onboarding
   - Displays: track count, albums, ISRCs, total Spotify streams
   - Per-track play counts with album art

4. **Onboarding Auth Fix**
   - Root cause: Next.js proxy doesn't forward cookies to backend
   - Fix: Use `getToken()` from Clerk + pass `Authorization: Bearer` header
   - All API calls now include auth token explicitly

5. **BigInt Serialization Fixes**
   - `Organization.totalStreams` (BigInt) broke JSON.stringify
   - Added `serializeOrg()` helper in orgRoutes.ts
   - `SpotifyCatalogTrack.playCount` converted to string in catalog response

### Security Fixes (Dex Review)
1. **Cross-org IDOR on catalog endpoints** — FIXED
   - `/spotify/artists/:id/catalog` now verifies `orgId` ownership
   - `/spotify/artists/:id/pull-catalog` same fix

2. **Cross-tenant artist collision** — FIXED
   - `linkSpotifyArtist()` was globally unique by spotifyId
   - Changed to `where: { spotifyId, orgId }` — each org gets their own copy
   - Schema: `@@unique([orgId, spotifyId])` replaces global unique

3. **Polling stale closure** — FIXED
   - CatalogPreview useEffect was checking stale state
   - Rewrote to use local `loaded` flag from fetch response
